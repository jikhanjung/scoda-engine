# 2026-04-16 | Version Bump to 0.3.6

## Summary

Docker Hub에 이미 `honestjung/scoda-server:0.3.6` 이미지가 배포된 상태였으나,
레포 내 버전이 0.3.5에 머물러 있어 불일치가 발생했다. 레포 버전을 0.3.6으로 동기화했다.

## Background

GitHub Actions `release.yml` 워크플로우가 `workflow_dispatch` 입력값(`version_tag`)을
Docker 이미지 태그로만 사용하고, 코드 내 버전(`pyproject.toml`, `__init__.py`)은 자동으로
갱신하지 않는 구조였다. 릴리스 시 `scripts/bump_version.py`를 먼저 실행하지 않으면
이미지 태그와 코드 버전이 불일치하게 된다.

## Changes

`scripts/bump_version.py 0.3.6` 실행:

- `pyproject.toml`: 0.3.5 → 0.3.6
- `scoda_engine/__init__.py`: 0.3.5 → 0.3.6
- `deploy/docker-compose.yml`: 0.3.5 → 0.3.6

## Notes

향후 릴리스 워크플로우에서 버전 범프를 자동화하거나,
릴리스 전 체크리스트에 `bump_version.py` 실행을 포함하는 것이 바람직하다.
